# plateau-space-accessibility-viewer
![Demo Status](https://github.com/ykkkatoh/plateau-space-accessibility-viewer/actions/workflows/keep_awake.yml/badge.svg)

PLATEAUの3D都市モデルを用いて、任意のエリアの「観光向け／地域向け」の空間特性を可視化・定量比較するStreamlitアプリです。東京大学工学部精密工学科卒業研究の一環で開発しました。

🔗 **デモ**: [[Streamlit Cloud URL](https://plateau-space-accessibility-viewer.streamlit.app/)]

![アプリイメージ](demo.png)

## 概要
観光客と地域住民では、同じ街であっても「実際に利用できる空間」が異なります。商業施設や公園は原則誰でも利用できますが、住宅地や農地は基本的に地域住民しか利用しません。

この違いを定量化したものが**補正係数α**です。
観光客が利用できる総面積に対する地域住民が利用できる総面積の比で、αが大きいほど地域住民向けの空間の割合が大きく、観光客が入り込んだ際の空間的負荷が高いエリアと解釈することができます。

### 補正係数αの背景
オーバーツーリズム（観光による悪影響が住民生活に及ぶ状況）を管理するには、観光が地域に与える影響を継続的に把握するモニタリングが重要です。
そのための指標として、「地域関係者数に対する観光客数の比」である観光強度（Tourism Intensity）が国などのマクロな規模で用いられています。
近年、GPSデータにより数百メートル四方のミクロスケールでの算出が可能になりましたが、ミクロ化すると住民の体感と一致しない場合があることが先行研究で示されています。

観光強度は人数の比で定義されていますが、その人数自体がエリアの空間用途の影響を受けています。
例えば商業ビルや公園が多いエリアには観光客・地域住民の双方が集まりますが、住宅地が多いエリアには地域住民しか集まりません。
ミクロスケールではこうした用途の偏りが顕著になるため、人数の比だけでは実態を捉えきれない場合があります。
そこで卒業研究では、空間用途を考慮した補正をかけることで実態との整合性を高めることを試みました。

具体的には、PLATEAUオープンデータの建物高さと土地利用用途から**補正係数α**を定義し、
観光強度に乗じることで住民体感との整合性が向上することを京都市東部のデータで検証しています。

詳細については以下を参照してください：
- 〔『観光と情報』査読論文〕（準備中）

## ビューアの機能
任意エリアにおける補正係数αの算出・可視化・比較をブラウザ上で行えます。

### 各モードの詳細
**比較モード**
- 地図上で矩形を描くだけで任意のエリアを指定（最大3エリア）
- 複数エリアの補正係数αを並べて比較
- 土地利用カテゴリの色分け表示（3D/2D切替）
- 建物クリックで用途・投影面積・推定延床面積・階数を表示

**詳細モード**
- 基準点・行数・列数・ボックスサイズを指定して分析グリッドを一括配置
- 補正係数αのヒートマップ表示
- 計算結果のCSVエクスポート

## セットアップ
### 1. リポジトリのクローン
```bash
git clone https://github.com/ykkkatoh/plateau-space-accessibility-viewer.git
cd plateau-space-accessibility-viewer
pip install -r requirements.txt
```

### 2. データの準備
対象都市の`bldg`（建物）と`luse`（土地利用）のCityGMLデータを[G空間情報センター](https://www.geospatial.jp/ckan/dataset/plateau)からダウンロードし、以下のディレクトリ構成で配置してください。
```
data/
├── bldg/        # 建物GeoJSON（*.geojson）
├── luse/        # 土地利用GeoPackage（*.gpkg）
├── landuse_name_map.csv       
├── landuse_category_roots.csv 
└── landuse_category_groups.csv 
```

CityGMLデータの取得・変換方法の詳細については以下も適宜参照してください：
- 〔『観光と情報』解説記事〕（準備中）

### 3. 起動
```bash
streamlit run app.py
```
